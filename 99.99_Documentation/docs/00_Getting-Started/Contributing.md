## Contributing

There's just a few simple rules you should follow whenever you're contributing
to ensure that changes are made smoothly without messing anything up. Follow
these steps:

1. Make an issue for whatever you're on.
2. Create a branch and assign it to that issue.
3. Checkout that branch on your machine and start working.
4. Make sure your changes are limited to one folder, plus a documentation folder
   if you added documentation.
5. When you're done submit a pull request.
6. Wait for someone to review your pull request.
7. Merge the request.
8. Celebrate!

This is the general rules for anything significant, if you're making a small
change avoid commiting to main and just make a branch really quickly and then
ask me to merge for it you.
